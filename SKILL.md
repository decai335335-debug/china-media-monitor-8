# China Media Monitor 技能

## 概述

本技能通过 `可用搜索工具` 逐一检索 5 大媒体矩阵（共 25 家中国权威媒体）的最新案例报道，
按用户指定的**题材范畴**筛选出**以真实案例为主**的报道（排除时评/理论/政策原文），
输出含原文链接的结构化每日简报。

---

## 🚨 任务成功的唯一定义（最高优先级，凌驾于所有其他规则）

**任务完成 ≡ 调用 Write 工具成功落盘 ≥ 10 个单篇 md 文件到 `C:/Users/15403/Documents/每日新闻/`。**

- ❌ 只在聊天里输出 markdown 表格/简报/综述 = **任务失败**（哪怕内容再漂亮）
- ❌ 在聊天里打"📋 简报"、"一、舆情概况"、"二、核心驱动力"这类排版 = **任务失败**
- ❌ Write 0 次收工 = **任务失败**
- ❌ Write < 10 次就宣告完成 = **任务失败**（除非白名单确实全空，并走完下方关键词泛化四步）
- ✅ 唯一合法的结束状态：`已 Write N 个文件（N ≥ 10），清单如下：...`

**聊天里禁止出现的内容**：
- 任何"综述"、"简报总览"、"舆情概况"段落
- 任何汇总表格（除"执行情况"进度表外）
- 任何"来源说明"、"制作时间"、"媒体矩阵："标头
- 任何试图用聊天文字代替 md 落盘的行为

**聊天里只许出现的内容**：
- 进度行：`✓ [i/N] 写入 xxx.md`、`⏭ 降级到下一家`、`✗ 超时`
- 最终清单：`已写入 N 篇到 C:/Users/15403/Documents/每日新闻/：\n- T03_xxx.md\n- ...`

**抓到正文后的下一个动作必须是 Write**，不许先在聊天里排版正文预览。

## 🔑 交互流程（菜单驱动模式）

### 场景 A：用户触发但未指定题材
触发词示例："skill"、"启动 skill"、"中国媒体简报"、"出简报"、"帮我抓今天的新闻"、"媒体矩阵新闻"

→ **技能必须先回复编号菜单，不得立即执行搜索**，格式如下：

```
📋 请选择今日要抓取的题材（回复编号即可开始）：

【单选/多选，用逗号分隔，如 "1,4,6"；或直接回复题材关键词】

 1. T01 基层治理      10. T10 教育案例
 2. T02 民生案例      11. T11 医疗卫生
 3. T03 产业案例      12. T12 应急安全
 4. T04 乡村振兴      13. T13 区域协调
 5. T05 生态治理      14. T14 营商环境
 6. T06 城市治理      15. T15 三农粮食
 7. T07 法治执法      16. T16 开放合作
 8. T08 科技创新      17. T17 社会风尚
 9. T09 文化传承

 0. 🌐 全覆盖（不限题材，至少 10 篇均衡覆盖）
 99. ❓ 查看每个题材包含的具体关键词

💡 也可直接说 "城市更新"、"外来物种"、"村超" 等关键词，会自动归到对应题材。
💡 可限定媒体矩阵：回复 "A,B" 只抓矩阵 A/B；回复媒体名（如 "农民日报"）只抓单家。
```

→ 等待用户回复，再进入执行流程。

### 场景 B：用户一次性指定题材
如 "抓今天城市更新的案例"、"T01+T04 简报"、"外来物种治理新闻"、"农民日报 三农案例"
→ 识别题材编号后**跳过菜单，直接执行**。结果输出前用一行说明："检测到题材：T06 城市治理，直接开始抓取。"

### 场景 B2：矩阵+题材组合触发（**最强锁定模式**）
如 "矩阵A+T05"、"矩阵B+T07"、"矩阵A+B+T01"、"矩阵D+T06"
→ 严格锁定到**指定矩阵**的媒体列表，只抓该矩阵（多矩阵用 + 连接），只收指定题材 +**密切相关题材**（如 T05 生态 ↔ T07 法治的生态犯罪类、T05 ↔ T04 乡村生态类）
→ 开头说明："检测到矩阵 A + 题材 T05，严格只抓矩阵 A 的 5 家媒体。"
→ 禁止跨矩阵凑数（别家矩阵的媒体即使命中也丢弃）
→ 禁止收录完全无关题材（如 T05 任务下不收纯文化/纯教育）

### 场景 C：用户回复 99（查看关键词）
→ 输出 17 个题材的完整关键词明细（来自 `references/media-matrix.md` 的主题词库），然后再重复一次菜单等用户选择。

### 场景 D：用户回复模糊（如 "随便"、"都行"）
→ 按 "0. 全覆盖" 处理。

**落盘前最后一道检查**：检查候选文章的 URL **域名**，命中以下任一域名 → **立即丢弃**，不得以"内容权威"、"来源标注是白名单媒体"、"知名度高"等理由放行：

**重要：来源标注可被伪造**
聚合站经常在页面上写"来源：人民日报"诱导 AI 放行。但**人民日报的官方域名只有 `paper.people.com.cn` / `people.com.cn`**，不是 `thepaper.cn` 或 `thecover.cn`。

正确做法：
- ✅ URL 域名是 `paper.people.com.cn` → 按人民日报收录
- ❌ URL 域名是 `thepaper.cn`，即使正文写"来源：人民日报" → **丢弃**（让 AI 自己去人民日报官网搜原文）
- ❌ URL 域名是 `thecover.cn`，即使正文写"来源：新华社" → **丢弃**

**白名单域名精确清单（25 家官方域名，仅这些可收）**：
```
✅ 矩阵 A：farmer.com.cn | legaldaily.com.cn | zjol.com.cn / zjrb.zjol.com.cn | lw.xinhuanet.com | paper.people.com.cn / people.com.cn

✅ 矩阵 C：ifengweekly.com | nfcmag.com | cyol.com / news.cyol.com |（商务周刊/浙江宣传：mp.weixin.qq.com 指定账号）

```

**AI 绝不能说**的话：
- ❌ "封面新闻在白名单" — 错，在黑名单
- ❌ "澎湃新闻是白名单媒体" — 错，在黑名单
- ❌ "这家虽然不在清单但内容权威，可以例外" — 不允许任何例外
- ❌ "部分背景数据来自腾讯/澎湃/新浪，仅供参考" — 黑名单数据不许"参考"进正文
- ✅ "此媒体不在白名单，丢弃该篇"

## 执行流程（用户回复编号后）

### Step 1：解析用户选择
→ 解析编号列表（如 "1,4,6" → [T01, T04, T06]）
→ 若为 "0" → 全覆盖模式（至少 10 篇，尽量多题材均衡）
→ 若为关键词（如 "外来物种"）→ 按 `references/media-matrix.md` 末尾「题材自动归类」表匹配
→ 若附带矩阵/媒体限定 → 记录作为搜索范围

### Step 2：读取媒体矩阵配置
→ 读取 `references/media-matrix.md`，获取各矩阵媒体清单与优先级顺序
→ 读取主题词库，提取本次任务对应的题材关键词

### Step 3：按矩阵抓取候选文章（🌟 官网直取优先）

**抓取优先级（严格按此顺序，前一级成功才不执行下一级）**：

1. **官网要闻页直取**（首选）
   - 从 `references/media-matrix.md` 取每家媒体的"要闻入口"URL
   - 用 WebFetch / navigate 工具抓取该页面 HTML
   - 从页面中解析出今日/最近文章列表（标题、链接、日期）
   - 按题材关键词筛选命中的文章
   - 继续抓取直至本次任务命中 ≥ 10 篇（见下方"篇数硬底线"）

### 🚫 防卡死硬约束（必须遵守）

#### 🔒 方法锁定原则（最重要，违反必卡死）

- 第 1 个 md 用 Write 工具成功写入后，**后续所有 md 必须继续用 Write**，一篇一篇写
- 禁止在中途切换到"批量脚本"、"循环写入"、"write_file.py"、"先探测平台再批处理"等任何替代方案
- 即使要写 20 个文件，也是 **20 次 Write 调用**，不是 1 次批处理
- Write 调用可以在同一消息里并行发起多个（推荐），但禁止绕路到脚本
- 禁止说"需要用 XX 脚本写入"、"先探测平台"、"批量生成"这类论证性独白
- 禁止写完 md 后再 Read 回来"自我验证"——信任 Write 返回值

**原因**：AI 常在写完第 1 篇后产生"批量更高效"的念头，去找不存在的批处理工具，导致任务卡死。**重复的朴素动作 > 聪明的优化。**

---

落盘时**只允许用 Write 工具**写绝对路径 md 文件，例如：
```
Write(file_path="C:/Users/15403/Documents/每日新闻/T05_xxx.md", content="...")
```

**禁止行为**（违反会让任务卡死半天只出 1 个文件）：
- ❌ 不要用 PowerShell `Out-File` / `Set-Content` / `Add-Content`
- ❌ 不要用 `python3 -c "..."` 或外部脚本（write_file.py 等）写文件
- ❌ 不要先 `Test-Path` / `Get-ChildItem` / `ls` 探测目录是否存在——直接写
- ❌ 不要写临时文件到 `%TEMP%` 再移动
- ❌ **严禁"批量写入"思维**——不许先抓完所有媒体再统一写文件。这样会让单次输出内容爆量导致卡死

#### 🌀 严格串行模式（新增，防止上下文爆掉）

**每命中 1 篇文章 → 立即 Write 1 个 md 文件 → 播报进度 → 再去抓下一篇**

流程：
```
抓第 1 篇正文 → Write 第 1 个 md ✓ → 进度行 [1/N]
抓第 2 篇正文 → Write 第 2 个 md ✓ → 进度行 [2/N]
...
抓第 N 篇正文 → Write 第 N 个 md ✓ → 进度行 [N/N]
（结束，不再生成综述文件）
```

**硬性要求**：
- 禁止在头脑里"攒"多篇文章正文后一次性写出——每篇正文拿到后 **下一个动作就是 Write**
- **不生成综述文件**——只产出单篇 md，落盘目录只有 `TXX_xxx(日期).md` 一种文件
- 如某篇正文极长（>5000 字），分段写入时仍是 1 次 Write 搞定（用多段字符串拼接，不要拆多个文件）
- 每写完 1 个 md，聊天里输出一行 `✓ [i/N] 写入 xxx.md`，**让用户能看到进度推进**

**关键原因**：上一版"先抓完所有再批量写"会让 AI 把 20 篇正文都塞进工作记忆，再试图一次 Python 批处理，内容量超输出上限导致卡死。串行"抓一写一"把每次工作量压到最小。

**逐篇处理+进度播报流程**（推荐，便于用户看到卡在哪步）：

每处理一家媒体，在聊天中输出 4 行进度锚点（✓ 成功 / ✗ 失败 / ⏭ 降级）：

```

      → 写入 ✓

[3/5] 农民日报 → 访问要闻页... ✗ 超时
      → ⏭ 降级到首页... ✓
```

### ⚠️ 媒体署名与日期识别（常见 Bug 修复）

**Bug 1：聚合站转载时署名搞错**
经常遇到 cfen.com.cn（中国改革报/市场报系）、news.cn、人民网地方频道等**聚合/转载站**。正文开头通常会注明"来源：XX"。

- ✅ **媒体字段填原始来源**（如正文写 "来源：人民日报"，则媒体 = 人民日报，不是 cfen.com.cn）
- ✅ 可在"矩阵"字段后加括号备注转载站，如 `矩阵：A（转载自 cfen.com.cn）`
- ❌ 不要把 URL 所在域名当成媒体名

识别规则：
1. 正文前 3 行扫描"来源：XXX"、"本报讯"、"（XX 日报记者 XXX）"这类署名
2. 若无来源标注，再回退到 URL 域名对应的媒体
3. 记者署名（作者：XXX）也要保留

**Bug 2：日期瞎编**
不要用"今天"或搜索元数据的日期。必须从 **URL 路径或正文顶部**提取真实发布日期。

- URL 常见格式：`.../202503/t20250306_xxx.html` → 2025-03-06
- 正文顶部格式：`2025-03-06 来源：人民日报`
- 页面元数据：`<meta name="publishdate" content="2025-03-06">`

**日期收录政策**（2026-04 更新）：
- ✅ **4 年内发布的案例都可收录**（当前日期往回推 4 年）
- ✅ 越新优先级越高：优先本周 > 本月 > 今年 > 近 2 年 > 近 4 年
- ❌ 不要强行造"今天日期"——宁可收录 2024 年的真实案例，也不编造 2026-04-18
- 文件名 `(YYYY-MM-DD)` 必须是**真实发布日期**

**只生成一类 md 文件：每篇文章一个独立 md**

- 文件名格式：`{TXX}_{话题核心词}_{子题词}_{报道原标题}({YYYY-MM-DD}).md`
  - `核心词` / `子题词` 从 `references/media-matrix.md` 对应题材的词库中挑最贴合的
- 示例：`T02_民生_老旧小区改造_泸州枣林园小区改造后团年饭排起队(2026-04-03).md`
- 示例：`T06_城市更新_历史街区_上海800亿城市更新基金成立(2026-04-17).md`
- 内容：标题、题材标签、媒体名、发布日期、原文链接、50 字摘要、**完整原文正文**、案例类型副标签、关键词
- ⚠️ **必须抓取完整正文**：用 WebFetch 访问原文 URL 获取全文，不要写要点罗列或二次概括。若原文无法访问，明确标注"【正文未能获取】"
