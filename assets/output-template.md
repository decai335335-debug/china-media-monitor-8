## 文件类型 1：单篇文章 md

### 文件名
```
{TXX}_{话题核心词}_{子题词}_{报道原标题}({YYYY-MM-DD}).md
```

- `{TXX}`：题材编号，如 T02、T06
- `{话题核心词}`：从 `references/media-matrix.md` 对应题材的【核心词】中挑 1 个最贴合的（如 T02 的"民生"/"惠民"，T05 的"生态"/"绿色"，T06 的"城市治理"/"城市更新"）
- `{子题词}`：从对应题材的【子题词/机制词/场景词】中挑 1 个该文章最命中的（如"老旧小区改造"、"田长制"、"村寨银行"、"一枚印章管审批"）
- `{报道原标题}`：原文标题，非法字符按下方规则替换
- `{YYYY-MM-DD}`：报道发布日期

**示例**：
- `T02_民生_老旧小区改造_泸州枣林园小区改造后团年饭排起队(2026-04-03).md`
- `T02_惠民_适老化改造_半月谈适老化改造改到老人心坎里(2026-04-14).md`
- `T06_城市更新_历史街区_上海800亿城市更新基金成立(2026-04-17).md`
- `T05_生态_两山转化_X县稀土矿区生态+光伏模式落地(2026-04-10).md`
- `T15_粮食安全_田长制_某省田长制数字化闭环巡查(2026-04-12).md`

### 文件内容模板

```markdown
# {报道原标题}

- **题材**：{TXX} {题材名称}（如 T02 民生案例）
- **副标签**：{政府试点 / 基层创新 / 民生落地 / 调查报道 / 企业实践 / 跨域协作 / 数据亮眼}
- **媒体**：{媒体名称}
- **矩阵**：{A/B/C/D/E}
- **发布日期**：{YYYY-MM-DD}
- **原文链接**：{完整 URL}

## 摘要（50 字内）
{谁/在哪/做了什么/结果如何}

## 正文

{**完整原文正文**，不是摘要、不是要点罗列。保留原文段落结构、具体数据、人物名、地点、引语。}

{获取方式：用 WebFetch / 浏览器 navigate 工具访问原文 URL 抓取正文，清洗掉导航/广告/相关推荐等非正文内容。若原文超长，完整保留不截断。}

{若无法获取正文（链接失效、付费墙、反爬），在此处写："【正文未能获取】" 并说明原因，同时保留搜索摘要作为备份。}

## 关键词
{从 media-matrix.md 主题词库中选 3-5 个命中词}

---
*抓取日期：{YYYY-MM-DD}*
*来自 china-media-monitor 技能*
```

## ⚠️ 综述文件已移除

本技能 **不再生成综述文件**。落盘目录只有单篇 md（文件类型 1），不要创建任何 `综述_*.md`。

## 文件命名合规处理

### 非法字符替换表（Windows 文件系统）

| 原字符 | 替换为 |
|--------|--------|
| `\` `/` `:` | `_` |
| `*` `?` | 删除 |
| `"` | `'` |
| `<` `>` | 全角 `＜` `＞` |
| `\|` | `_` |
| 换行 | 空格 |
| 连续空格 | 单个下划线 |

### 长度处理
- 文件名（不含路径）控制在 150 字符以内
- 标题超长（>80 字）截断至 80 字 + `...`

---

## 多题材任务的文件组织

用户一次选多个题材（如 T01+T04+T06）：
- 单篇文章按各自题材编号命名：`T01_xxx.md`、`T06_xxx.md`
- **不生成综述**——只按题材编号把单篇 md 放在同一目录即可

## 全覆盖模式

用户选 "0" 时：
- 单篇按各自题材命名 `T0X_xxx.md`
- 目标 ≥ 10 篇，题材尽量均衡，**无综述**
