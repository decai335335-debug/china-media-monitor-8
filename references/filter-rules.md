# 文章筛选规则

## 核心判断：这篇文章是「案例报道」还是「时评/理论/政策原文」？

本技能**只保留案例报道**，排除所有评论、社论、政策文件、理论综述、展望预测。
判断标准与题材无关——无论是基层治理、城市更新、外来物种治理还是科技创新，
都按同一套规则筛选。

---

## ✅ 保留：案例报道（满足以下任意 2 条即保留）

### 判断指标

| 指标 | 说明 | 示例信号词 |
|------|------|-----------|
| **有具体主体** | 有真实企业/机构/地区/个人名称 | "XX 村"、"XX 市"、"XX 公司"、"XX 医院"、"XX 执法大队" |
| **有时间节点** | 有明确的事件时间 | "近日"、"本月"、"2026 年 X 月"、"日前" |
| **有数据支撑** | 有具体数字描述结果 | "整治 500 亩"、"效率提升 30%"、"节省成本 X 万"、"办结 X 件" |
| **有落地场景** | 描述具体应用场景 | "用于"、"部署在"、"投入使用"、"在 XX 试点" |
| **有结果描述** | 描述项目成效或进展 | "已实现"、"成功落地"、"正在推进"、"取得成效" |
| **有调查采访** | 有记者采访或现场记录 | "记者走访"、"采访中了解到"、"在现场看到" |

### 典型保留类型（按题材示例）

- **基层治理**："XX 市推行网格化管理，XX 社区矛盾化解率提升 X%"
- **城市更新**："XX 老旧小区改造完成，加装电梯 X 部，居民反馈……"
- **生态治理**："XX 县红火蚁防治取得阶段性成效，清理 X 亩"
- **法治执法**："XX 市推行'一支队伍管执法'，首违不罚案例 X 起"
- **民生案例**："XX 市适老化改造试点，X 户老人受益"
- **乡村振兴**："XX 村发展'一村一品'，村集体收入 X 万元"
- **产业案例**："XX 企业入选专精特新'小巨人'，研发投入占比 X%"
- **科技创新**：企业产品落地、首台套突破、技术应用场景
- **文化传承**："XX 非遗项目走进校园"、"XX 传统村落保护修缮"
- **调查报道 / 深度报道**：记者走访 + 多个具体场景

---

## ❌ 排除：时评/理论/政策文章（满足以下任意 1 条即排除）

### 判断指标

| 指标 | 说明 | 示例信号词 |
|------|------|-----------|
| **无具体主体** | 泛泛而谈，无具体名称 | "企业应当"、"社会需要"、"各地要"、"基层干部应" |
| **纯观点表达** | 全篇为作者/专家观点 | "笔者认为"、"专家指出"、"有观点认为" |
| **政策原文转载** | 直接引用政策文件内容 | "根据 XX 文件"、"依据规定"、"《XX 办法》明确" |
| **号召式结尾** | 以倡议/呼吁结尾无案例 | "我们应该"、"必须坚持"、"要……要……要……" |
| **署名为评论员** | 文章类型为评论/社论 | "评论员文章"、"本报评论"、"社论"、"编者按" |
| **展望预测** | 通篇描述未来可能 | "未来将"、"有望"、"前景广阔" |

### 典型排除类型

- 评论员文章、社论、编者按
- 专家访谈（仅有观点，无落地案例）
- 政策解读（仅解读文件，无实际项目）
- 学术性综述（引用论文，无新闻事实）
- 展望预测类（"未来 AI 将……"、"乡村振兴应当……"）
- 领导讲话全文转载
- 空泛的"经验总结"稿（无具体地名/数据）

## 🔑 媒体原始来源识别（避免署名错误）

聚合/转载站点常见：cfen.com.cn（中国改革报）、news.cn（新华网）、people.com.cn 各地方频道、央视网转载稿等。

### 识别流程
1. **优先扫描正文前 3 行**的署名标注：
   - "来源：人民日报"、"来源：新华社" → 媒体 = 人民日报 / 新华社
   - "本报讯"、"（X 日报记者 XXX 报道）" → 该报即原始媒体
   - "作者：XXX" → 保留为记者署名字段
2. **次选：页面元数据** `<meta name="source">` 或 `<span class="source">`
3. **兜底：URL 域名** 仅在前两步都没结果时使用

### 输出字段要求
- 媒体：**原始来源**（非聚合站）
- 矩阵：按原始媒体所属矩阵分类（不按 URL 域名分）
- 可选备注：`（转载自 cfen.com.cn）`

## 🗓 日期识别与收录范围（2026-04 更新）

### 日期提取（必须真实，不得编造）
按优先级：
1. URL 路径日期：`/202503/t20250306_xxx.html` → 2025-03-06
2. 正文顶部：`2025-03-06 来源：XXX`
3. 页面元数据：`<meta name="publishdate">` / `<meta name="og:published_time">`
4. 如上述全失败，标注 `[日期待核实]`，不要瞎写

### 收录范围
- ✅ **4 年内案例均可收录**（当前日期往回推 4 年）
- 优先级：本周 > 本月 > 本季度 > 今年 > 近 2 年 > 近 4 年
- ❌ 禁止把陈年案例标成"今天"——文件名日期必须是真实发布日期
- 若一家媒体近期没有命中文章，回溯 4 年内的旧案例优于降级到下一家媒体

## 边界情况处理

### 情况 1：政策 + 案例混合文章
**处理**：若文章后半部分包含至少 1 个具体落地案例 → **保留**
**标注**：题材标签后加注「政策+案例」

### 情况 2：专家访谈含案例
**处理**：若专家提及具体项目名称、数据、地点 → **保留**
**标注**：加注「访谈案例」

### 情况 3：调查报道无结论
**处理**：只要有多个具体采访对象和场景描述 → **保留**

### 情况 4：标题含"案例"但内文无实质
**处理**：以内文判断为准，标题不作参考 → **排除**

### 情况 5：一文多题材
**处理**：以主叙述线对应的题材为准，次要题材可在标签中加副标签
**示例**：既讲城市更新又讲基层治理 → 主标签 T06，副标签 T01

## 快速判断流程图

```
文章进入筛选
     ↓
是否有具体企业/机构/地区名称？
     ├── 否 → 大概率排除，继续验证
     └── 是 → 继续下一步
          ↓
     是否描述了具体事件（做了什么/结果如何）？
          ├── 否 → 排除
          └── 是 → 进入保留候选
               ↓
          是否有评论员/社论/社评标识？
               ├── 是 → 排除
               └── 否 → 继续
                    ↓
               是否至少满足保留指标中的 2 条？
                    ├── 否 → 排除
                    └── 是 → ✅ 保留，进入题材归类
```

## 题材与案例标签分类（输出时使用）

### 题材主标签（必选 1 个，T01~T17）
对应 `references/media-matrix.md` 的 17 大题材。

### 案例类型副标签（可选 1~2 个）

| 标签 | 适用场景 |
|------|---------|
| `政府试点` | 政府机构推行的改革/试点项目 |
| `企业实践` | 企业主导的项目/产品落地 |
| `基层创新` | 乡镇/社区/村级的首创做法 |
| `民生落地` | 直接面向居民/个人的应用 |
| `调查报道` | 记者深度调查/走访类 |
| `访谈案例` | 含具体案例的人物访谈 |
| `政策+案例` | 政策背景下的落地案例 |
| `跨域协作` | 跨部门/跨地区协作的案例 |
| `数据亮眼` | 含突出数据成效的报道 |
